# translate

프론트엔드 개선 작업 시 새로 추가된 모든 사용자 대면 텍스트에 대해 한국어와 영어 i18n 항목을 추가합니다.
모든 프론트엔드를 뒤져서 개선하는 것이 아니며, 사용자가 요청한 영역 혹은 가장 최근 사용자의 명령에 의해 수정한 파일만 확인 합니다. 

## i18n 구조

**중요**: 영어가 메인 언어이며, **키 자체가 영어 문장**입니다.

### en-US/translation.json (영어)
```json
{
  "Avatar ready!": "",
  "Initializing avatar...": "",
  "Save Settings": ""
}
```
- **Key**: 영어 문장 (코드에서 사용)
- **Value**: 빈 문자열 `""` (키가 곧 표시될 텍스트)

### ko-KR/translation.json (한국어)
```json
{
  "Avatar ready!": "아바타 준비 완료!",
  "Initializing avatar...": "아바타 초기화 중...",
  "Save Settings": "설정 저장"
}
```
- **Key**: 동일한 영어 문장
- **Value**: 한국어 번역

## 작업 순서

1. **하드코딩된 텍스트 찾기**
   - 버튼, Toast, 폼 레이블, 오류 메시지 등

2. **$i18n.t() 호출로 변경**
   ```svelte
   <!-- Before -->
   <button>Save Settings</button>
   
   <!-- After -->
   <button>{$i18n.t('Save Settings')}</button>
   ```

3. **en-US에 추가** (값은 빈 문자열)
   ```json
   "Save Settings": ""
   ```

4. **ko-KR에 추가** (한국어 번역)
   ```json
   "Save Settings": "설정 저장"
   ```

## 규칙

### ✅ 해야 할 것
- 명확한 영어 문장을 키로 사용
- 문장 부호 포함 (!, ?, ...)
- 변수는 `{{variable}}` 형식 사용
- 신규 데이터는 가장 마지막에 넣을 것

### ❌ 하지 말아야 할 것
- 백엔드 로그는 번역 안 함
- 약어나 코드 스타일 키 사용 금지 (`init_avatar` ❌)
- 개발자용 디버그 메시지 제외
- 기존 항목의 순서를 변경하거나 알파벳 순으로 재정렬하지 말 것


## 체크리스트

- [ ] 사용자 대면 텍스트 → `$i18n.t()` 변환
- [ ] en-US 파일 **끝에** 영어 키 추가 (값 = "")
- [ ] ko-KR 파일 **끝에** 동일 키 + 한국어 번역 추가
- [ ] 기존 항목의 순서는 변경하지 않음
