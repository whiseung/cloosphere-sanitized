### Added

  - **다중 고객사 무중단 호환**: 신규 컬럼 `extension_engine_map` 이 비어 있는 동안에는 기존 단일 엔진 경로가 그대로 동작합니다. 관리자는 프로파일 카드의 "매핑으로 전환" 버튼으로 자기 시점에 옵트인하며, 자동 백필은 기존 엔진 + Native fallback 조합으로 회귀 0 을 보장합니다. 코드 롤백 시에도 legacy 컬럼 보존으로 즉시 복구됩니다.
  - **확장자 차단 정책**: 매핑이 활성화된 프로파일은 매핑에 없는 확장자 업로드를 HTTP 400 으로 차단해 관리자가 모르는 엔진 우회를 방지합니다.

### Fixed

- **📧 작업 로그 이메일**: 채널 미설정으로 발송되지 않았을 때 토스트가 success(녹색) 로 표시되던 문제를 warning(노란색) 으로 수정했습니다. 작업 로그 승인 메일은 항상 `support-cloosphere@cloocus.com` 으로 CC 사본이 전달됩니다 (모든 SMTP/SendGrid/MSGraph/Azure 발신자에 cc 파라미터 지원 추가).
- **📄 문서 추출 — 입력란 가시성 + Silent fallback 차단**: 문서 처리 프로파일 모달의 Document Intelligence Endpoint / Tika·Docling URL / Document AI 필드들이 배경/테두리 없는 raw input 으로 인해 화면에 안 보이던 문제를 공용 `Input` 컴포넌트 교체로 수정했습니다. 또한 추출 엔진이 지정됐는데 필수 자격증명이 비어 있을 때 조용히 내장 native 로더로 떨어지던 동작을 명시적 `ValueError` 로 실패하도록 변경 — 사용자가 외부 엔진을 우회한 줄 모르고 잘못된 추출 결과를 얻는 상황을 차단합니다.

## [1.0.3] - 2026-04

대규모 보안·거버넌스 릴리즈입니다. 엔터프라이즈 KMS (Azure Key Vault envelope, 분류별 KEK, 자동 회전, 변조 감지 감사 로그), MCP 도구 OAuth/SSO 인증과 화이트리스트, KG `kg_cypher` escape hatch 도구가 추가되었으며, 워크스페이스 공유 권한 모델이 완성되고 외부 IDP 통합도 강화되었습니다. 위 1.0.1 누적 항목에 이미 포함된 내용은 중복 기재하지 않습니다.

### Added

- **🔐 엔터프라이즈 KMS 통합**: 민감한 설정 값, DbSphere 연결 자격증명, 도구 연결 키, 사용자 API 키를 Azure Key Vault 의 외부 KEK 로 wrap 할 수 있게 되었습니다. AES-256-GCM 데이터 암호화 + RSA-OAEP-256 KEK wrap 구조이며, KEK 자체는 볼트를 벗어나지 않습니다. 관리자는 새 **Settings → 암호화** 탭에서 공급자를 설정하고, 일괄 마이그레이션과 tier 별 health-check 가 내장되어 있습니다.
  - **분류별 KEK 분리**: Restricted KEK URI 를 별도 지정하면 PII / 금융 분류가 그 KEK 로 wrap 되어, PII 만 crypto-shred 해도 다른 시크릿에 영향이 없습니다.
  - **자동 KEK 회전**: KV `rotation_policy` 가 새 KEK 버전을 만들면 스케줄러가 발견해 모든 envelope 을 새 KEK 로 재암호화합니다. 첫 활성화 시 Dry-run 모드로 "회전 예정" 을 감사 로그에만 기록하면서 검증할 수 있습니다.
  - **변조 감지 감사 로그**: 모든 wrap / unwrap / rotate / health_check / migrate 작업이 SHA-256 prev_hash + row_hash 체인으로 기록됩니다. 무결성 검증과 CSV 내보내기 (사유 필수) 가 새 **모니터링 → KMS 감사** 탭에 노출되며, 빠른 상태 블록은 Settings → 암호화 에 표시됩니다.
  - **원자적 저장(Atomic Save)**: 암호화 탭 저장 시 새 KEK 에 대해 자동 health-check 가 선행되며, KEK URI 가 변경되면 envelope 마이그레이션 여부를 묻는 다이얼로그가 표시됩니다 — 잘못된 URI 가 작동 중인 설정을 덮어쓰지 않습니다.
- **🤝 MCP 도구 OAuth / SSO 인증 + 화이트리스트**: 기존 인증 방식 (none / API key / bearer) 에 더해 OAuth (provider 별) 와 Cloosphere SSO 토큰 패스스루를 지원합니다. MCP 서버가 노출하는 도구 중 사용자에게 보여줄 도구를 개별 토글로 통제할 수 있어, 의도치 않은 도구 노출을 방지합니다.
- **🧠 `kg_cypher` 도구 — Read-only Cypher Escape Hatch**: 기존 5개 KG 도구로 표현하기 어려운 그래프 쿼리를 위해 read-only Cypher 를 직접 실행할 수 있는 도구가 추가되었습니다. 안전 실행 레이어가 read-only 를 강제 (mutating 쿼리 차단) 하고, LLM judge 가 쿼리 안전성을 한 번 더 검증하며, 결과는 `final_answer` 의 KG 컨텍스트로 자동 승격됩니다. Apache AGE `ORDER BY` alias 함정 가이드도 내장되어 첫 시도 정확도가 향상됩니다.
- **🧠 KG 시맨틱 메모리 5종**: `cypher_example`, `kg_schema_doc`, `kg_domain_doc`, `cypher_pattern`, `cypher_negative` 메모리가 KG 에이전트의 성공 패턴 재사용과 반복 실수 회피를 돕습니다. dedup·drift 처리도 내장되어 있습니다.
- **🤝 외부 IDP ID 토큰 패스스루 (Trusted Audiences)**: 외부 IDP(Entra, Google 등)가 발급한 ID 토큰을 신뢰 audience 화이트리스트로 직접 수용할 수 있어, 별도 토큰 교환 없이 기존 IDP 와 통합할 수 있습니다.
- **📅 스케줄 공유 권한 모델**: 스케줄에 read/write 2단계 access_control 공유가 추가되었습니다. read 공유자는 조회/복사, write 공유자는 수정/토글/즉시 실행이 가능합니다. 실행 컨텍스트는 항상 원 소유자로 고정되어 수동 트리거에 의한 권한 상승이 발생하지 않습니다.
- **📚 용어집 거버넌스 UX**: 단일 그룹 scope 라벨, 필터 칩, Copy 모달이 추가되어 read 공유자도 detail 진입과 복사가 가능하면서 수정 권한은 소유자/write 공유자로 유지됩니다.
- **🔐 워크스페이스 공유 권한 모델 일관화**: 도구·태그·용어집·스케줄에서 read 공유자의 detail 진입과 write 공유자에게만 보이는 수정 메뉴가 일관된 방식으로 정리되었습니다.
- **👤 사용자 편집 모달 개선**: 사용자 편집 화면에 소속 조직 단위가 표시되고 전체 UX 가 정리되었습니다.
- **🏷️ Tags Feature Permission**: workspace.tags read/write 권한이 추가되어 태그 관리 권한을 그룹 단위로 통제할 수 있습니다.
- **🔧 워커 큐 자동 정리**: Settings → 데이터 보존 에 좀비 / stuck 큐 메시지·컨슈머 자동 제거를 위한 임계값 설정이 추가되어, 백그라운드 워커를 수동 개입 없이 건강하게 유지할 수 있습니다.

### Changed

- **💬 인용 칩 디자인 개선**: 채팅 응답의 `[n]` 인용 칩이 가독성과 밀도가 개선된 디자인으로 재디자인되었습니다.
- **🤝 Teams 봇 멀티워커 세션 동기화**: Teams 봇 세션 state 가 Redis 로 공유되어 멀티워커 배포에서도 대화 컨텍스트가 보존됩니다.
- **🔧 멀티워커 환경 설정 동기화**: PersistentConfig / OAuth / KMS 설정 변경이 Redis pub/sub 으로 모든 워커에 즉시 전파되어, 관리자 편집 후 워커별 설정 불일치가 발생하지 않습니다.

### Fixed

- **🧠 KG 파이프라인 안정화**: KB 청크 추출이 AGE 그래프에 `doc_entity` 노드도 작성하도록 수정 (이전엔 SQL 만 채우고 그래프는 비어있었음), KG 문서 검색 결과가 파일 단위 source 로 분리되며, KG jobs API 응답에서 자격증명이 마스킹됩니다.
- **🔧 안정성 개선**: 마이그레이션이 idempotent 하게 동작하며 좀비 schema 검증이 추가되고, 멀티워커 환경에서 `KMSRouter` 가 설정 변경 시 자동 reload 되며, 감사 로그 체인이 SQL 레벨로 직렬화되어 워커 간 race 로 인한 chain 깨짐이 차단됩니다. 검색 설정 모달 저장 시 `.trim()` TypeError 해결, OpenAI 연결 마스킹 secret 복원이 URL 기준으로 변경, OAuth 콜백마다 사용자 조직 단위 meta 가 동기화되도록 수정되었습니다.

## [1.0.2] - 2026-04

1.0.1 누적 항목에서 다루지 않은 작은 변경을 모은 패치 릴리즈입니다.

### Added

- **🔍 지식 베이스 배치 작업 UX**: 추출 진행률 표시, 재추출, 배치 삭제가 지식 베이스 파일 작업에 추가되었습니다.
- **🔍 KB 필터 — 용어집 타입 + Redis Queue 파이프라인**: 지식 베이스 추출 필터에 용어집 타입이 추가되고 추출 작업이 Redis Queue 파이프라인으로 처리됩니다.
- **🔧 관리자 온보딩 토글**: 첫 로그인 온보딩 안내 표시 여부를 관리자가 토글할 수 있습니다.
- **📝 에이전트 도구 설명 누락 경고 배너**: 에이전트 편집기에서 연결된 도구의 description 이 비어있으면 경고 배너가 표시됩니다.
- **🌐 UI 컴포넌트 통일**: 관리자 Locales 입력, 사용량 제한 액션 셀렉터, Inquiry 모달 셀렉터가 공용 컴포넌트로 마이그레이션되어 동작이 일관됩니다.

### Fixed

- **🔧 안정성 개선**: `stream=false` 요청도 UnifiedAgent 로 라우팅되도록 수정, KG-only 에이전트에서 KbSphere/DbSphere 독립 도구가 노출되지 않도록 차단, 도구 호출 프롬프트 일반화 (도메인별 예시 제거).

## [1.0.1] - 2026-04

1.0.0 릴리즈 이후의 기능 확장과 안정화 업데이트입니다. Microsoft Teams 봇 통합, Google Workspace 조직 동기화, 임베드 위젯 확장, 지식 그래프 Phase 2 구조 개편, 용어집 고도화, 워크스페이스 공유 태그 도입이 포함됩니다.

### Added

- **🤝 Microsoft Teams 봇**: Teams 채팅에서 Cloosphere 에이전트와 직접 대화할 수 있는 공식 봇이 추가되었습니다. 에이전트 선택, 대화 영속, 다국어 지원과 관리자 UI를 통한 설정·배포를 제공합니다.
- **🏢 Google Workspace 조직 동기화**: Google Admin Directory API 기반으로 조직·부서·그룹을 자동 동기화합니다. 동기화 과정에서 멤버 상세 정보까지 저장되며, 대규모 조직에서도 병렬 처리와 재시도로 안정적으로 동작합니다.
- **💬 임베드 위젯 확장**:
  - **게스트 모드**: 비로그인 사용자도 임베드 위젯을 사용할 수 있어 공개 웹사이트 대상 챗봇 배포가 가능합니다.
  - **SSO 토큰 교환**: 외부 인증 시스템이 발급한 토큰을 Cloosphere 세션으로 안전하게 교환하는 엔드포인트가 추가되었습니다.
  - **하단 사이드 패널 모드**: 페이지 하단 고정 패널 형태로 표시되는 새 디스플레이 모드가 추가되었습니다.
  - **사용자 리사이즈 + 호스트 레이아웃 자동 조정**: 사용자가 위젯 크기를 조절하면 호스트 페이지 본문이 자동으로 밀려 가려지지 않습니다.
  - **관리자 SSO 탭 + 디자인 옵션 확장**: SSO 설정 전용 관리 탭과 더 세밀한 디자인 옵션이 제공됩니다.
- **🧠 지식 그래프 Phase 2 — 문서 구조 확장**:
  - **컨테이너/문서/청크 노드 계층**: 지식 베이스의 문서 구조가 그래프에 체계적으로 반영됩니다.
  - **문서 타입 자동 분류**: 청크 내용 기반으로 문서 타입을 자동 식별하여 KB 필터로 활용할 수 있습니다.
  - **엣지 카탈로그 모달**: 엣지 타입을 카테고리별로 관리·추천하는 전용 모달과 설정 화면이 제공됩니다.
  - **그래프 전체화면 뷰 + 엣지 리스트**: 대규모 그래프 탐색을 위한 전체화면 뷰와 엣지 리스트 화면이 추가되었습니다.
  - **Graph-RAG 재설계**: KG 에이전트가 DbSphere 메모리와 연동되어 더 정확한 답변을 생성합니다.
  - **글로벌 진행률 표시**: KG 동기화 작업의 진행 상황이 상단 알림 센터와 전역 indicator로 실시간 표시됩니다.
- **📚 용어집 고도화**:
  - **카테고리 관리**: 용어집을 카테고리로 구분하고 관리할 수 있습니다.
  - **DB 값 추출 백그라운드화**: 대량 추출이 백그라운드로 실행되며, 추출된 용어를 인라인으로 검토할 수 있습니다.
  - **항목별 커스텀 지시사항**: 추출 항목별로 LLM에 전달할 지시사항을 개별 설정할 수 있습니다.
  - **추출 소스 선택**: 제목·본문·앞/뒤 N자 등 원하는 위치에서 용어를 추출할 수 있습니다.

## [1.0.0] - 2026-04 🎉 프로덕션 릴리즈

Cloosphere의 첫 정식 릴리즈입니다. 그동안의 모든 기능을 안정화하고, 용어집·데이터베이스·지식 베이스를 하나로 연결해 에이전트의 이해력을 끌어올리는 Knowledge Graph, 외부 사이트에 임베드 가능한 채팅 위젯, 대규모 파일 처리를 위한 큐 기반 워커 아키텍처가 추가되었습니다.

### Added

- **🧠 지식 그래프 (Knowledge Graph)**: 용어집, DbSphere, 지식 베이스를 하나의 그래프로 연결해, 에이전트가 "강남지점 VIP 고객의 월간 매출"처럼 복잡한 비즈니스 질문도 실제 데이터로 답할 수 있게 해주는 통합 지식 구조입니다.
  - **세 가지 정보원 통합**: 용어집의 비즈니스 용어, DbSphere의 테이블/컬럼/FK, 지식 베이스의 문서가 하나의 그래프로 통합되며 노드와 엣지가 자동 생성됩니다.
  - **자동 스키마 동기화**: DbSphere에서 가져온 테이블·컬럼이 노드로, 외래 키 관계가 엣지로 자동 반영됩니다. 기존 DbSphere 스키마 추출 결과를 재사용합니다.
  - **LLM 기반 엔티티 추출**: 연결된 지식 베이스 문서를 LLM이 분석해 엔티티와 관계(`produces`, `owned_by`, `located_in`, `has_risk` 등)를 추출하므로, 데이터와 문서를 가로질러 추론이 가능합니다.
  - **비즈니스 용어 → 컬럼 자동 매핑**: 용어집의 용어가 데이터베이스 컬럼과 필터식으로 매핑되어 NL→SQL 정확도가 크게 향상됩니다 ("VIP 고객" → `tier = 'VIP'`).
  - **후보 용어 검수**: 데이터베이스 디멘션 값에서 추출한 용어 후보가 검수 큐에 쌓이며, 수락 시 선택한 용어집에 자동 등록되고 `maps_to` 엣지까지 생성됩니다.
  - **지식 연결 (Knowledge Link)**: 데이터베이스의 디멘션 값(제품ID, 협력사 코드 등)을 지식 베이스 문서와 자동 매칭해, 데이터 행과 설명 문서를 직접 연결합니다.
  - **에이전트 연동**: 에이전트에 KG를 연결하면 `kg_resolve_term`, `kg_explore_context`, `kg_search_concepts`, `kg_find_related_tables`, `kg_neighbors` 5개 도구가 자동 제공되고, KG에 연결된 용어집·DbSphere·지식 베이스가 에이전트에 상속됩니다.
  - **그래프 시각화**: Cytoscape 기반의 인터랙티브 그래프로 노드와 엣지를 시각적으로 탐색할 수 있고, 노드 타입별 필터와 검색을 제공합니다.
  - **잡 추적 & 진행률**: 장시간 걸리는 동기화와 엔티티 추출 작업의 진행률과 오류 내용이 실시간으로 표시되며, UI에서 취소할 수 있습니다.
- **💬 임베드 채팅 위젯**: 외부 웹사이트에 Cloosphere AI 채팅을 손쉽게 심을 수 있는 임베드 위젯이 추가되었습니다. 관리자 설정에서 위젯을 생성하고 디자인을 자유롭게 커스터마이징한 후, 한 줄의 스크립트 태그로 어디든 붙일 수 있습니다.
  - **다양한 표시 모드**: 우측 하단 버블, 좌/우 사이드 패널, 인라인 삽입, 전체 화면 등 5가지 모드를 지원합니다. 외부 사이트의 디자인에 자연스럽게 어울리도록 선택할 수 있습니다.
  - **풍부한 디자인 편집기**: 색상(배경, 헤더, 메시지, 전송 버튼), 헤더 텍스트, 위젯 너비/높이, 위치, 버블 아이콘과 전송 버튼 아이콘(7종 프리셋 + 직접 업로드)을 라이브 미리보기로 즉시 확인하며 조정할 수 있습니다.
  - **자체 로그인 지원**: 위젯이 토큰 없이 로드되면 OAuth(Microsoft, Google, GitHub 등)와 이메일/비밀번호 로그인 화면을 자동 표시합니다. 외부 서비스가 발급한 토큰을 직접 전달하는 방식도 지원합니다.
  - **임베드 페이지 직접 조작**: AI 에이전트가 사용자가 보고 있는 페이지의 폼을 직접 채우거나 버튼을 클릭하는 인터랙티브 작업이 가능합니다. 그룹웨어, 신청서, 데이터 입력 화면 등에 적용하면 사용자가 자연어로 요청한 내용을 AI가 화면에서 직접 처리합니다.
  - **도메인 보안**: 위젯별로 허용 도메인을 지정할 수 있어, 의도치 않은 사이트에서 사용되는 것을 방지합니다.
- **⚙️ 안정적인 대용량 파일 처리 (큐 기반 워커)**: 지식기반에 대량의 파일을 업로드해도 메인 서비스가 멈추지 않도록, 파일 처리 작업이 큐 기반 워커에서 순차적으로 안정 처리됩니다.
  - **재시도 + 실패 표시**: 일시적 오류는 자동 재시도되고, 실패한 파일은 사용자가 화면에서 확인하고 재시도할 수 있습니다.
  - **동일 파일명 교체**: 같은 이름의 파일을 다시 업로드하면 기존 파일을 자동으로 교체합니다.
  - **파일 처리 현황 카드**: 진행 중/완료/실패 상태를 한눈에 보여주는 알림 카드를 제공합니다.
