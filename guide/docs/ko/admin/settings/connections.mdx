## OpenAI API 연결

OpenAI 또는 OpenAI 호환 API(Azure OpenAI, AI Foundry 등)와 연결합니다. 여러 연결을 동시에 구성하여 부하를 분산할 수 있습니다.

<Steps>
  <Step title="연결 추가">
    OpenAI API 섹션의 **"+" 버튼**을 클릭합니다.
  </Step>
  <Step title="연결 정보 입력">
    연결 Modal에서 API Base URL, API Key를 입력합니다.
  </Step>
  <Step title="연결 확인">
    저장 후 모델 목록이 자동으로 갱신되는지 확인합니다.
  </Step>
</Steps>

<Tabs>
  <Tab title="OpenAI">
    | 설정 항목 | 설명 |
    |-----------|------|
    | **API Base URL** | `https://api.openai.com/v1` |
    | **API Key** | OpenAI에서 발급받은 키 (`sk-...`) |
    | **Prefix ID** | 모델 ID 접두사 (선택 — 다중 연결 시 충돌 방지) |
    | **Model IDs** | 수동 지정 (비우면 자동 조회) |
  </Tab>
  <Tab title="Azure OpenAI">
    | 설정 항목 | 설명 |
    |-----------|------|
    | **API Base URL** | `https://{리소스명}.openai.azure.com/` |
    | **API Key** | Azure Portal에서 발급받은 키 |
    | **API Version** | API 버전 (필수, 예: `2024-02-15-preview`) |
    | **Prefix ID** | 모델 ID 접두사 (선택) |
    | **Model IDs** | 배포된 모델명 수동 지정 |
  </Tab>
  <Tab title="AI Foundry">
    | 설정 항목 | 설명 |
    |-----------|------|
    | **API Base URL** | `https://{프로젝트명}.{리전}.models.ai.azure.com/` |
    | **API Key** | AI Foundry에서 발급받은 키 |
    | **API Version** | API 버전 (기본: `2024-05-01-preview`) |
    | **Prefix ID** | 모델 ID 접두사 (선택) |
    | **Model IDs** | 수동 지정 (비우면 연결 확인 시 자동 조회) |

    <Accordion title="Azure OpenAI와 AI Foundry의 차이">
      | | Azure OpenAI | AI Foundry |
      |---|---|---|
      | **엔드포인트 형식** | `{url}/openai/deployments/{모델}/...` | `{url}/openai/v1/...` |
      | **모델 식별** | 배포(deployment) 이름 | 모델 ID |
      | **인증** | API Key 또는 Bearer Token | Bearer Token만 |
      | **용도** | Azure에서 OpenAI 모델 전용 | Azure AI에 배포된 다양한 모델 (OpenAI + OSS) |

      Azure OpenAI는 모델을 **배포 단위로** 관리하고, AI Foundry는 **모델 카탈로그**에서 바로 접근합니다. 이미 Azure OpenAI를 사용 중이라면 **별도로 AI Foundry를 추가할 필요는 없습니다** — OSS 모델이나 AI Foundry 전용 모델이 필요할 때 추가하세요.
    </Accordion>
  </Tab>
  <Tab title="Vertex AI">
    | 설정 항목 | 설명 |
    |-----------|------|
    | **Project ID** | Google Cloud 프로젝트 ID (필수) |
    | **Location** | 리전 (기본: `us-central1`) |
    | **인증** | Global GCP Key 또는 Custom Service Account JSON |

    연결 확인 시 사용 가능한 모델 목록이 자동으로 표시됩니다.
  </Tab>
</Tabs>

### 연결 설정 공통 사항

각 연결에는 다음 공통 기능이 있습니다:

| 기능 | 설명 |
|------|------|
| **연결 확인** | 설정 모달의 확인 버튼으로 API 접속 테스트 |
| **Prefix ID** | 여러 연결의 모델명이 겹칠 때 `{prefix}.{model}` 형태로 구분 |
| **활성화/비활성화** | 연결별 토글로 일시 중지 가능 (삭제하지 않고) |
| **Model IDs** | 수동 지정 시 해당 모델만 노출. 비우면 API에서 자동 조회 |

<Tip>
  동일 프로바이더에 **여러 연결**을 추가하면 모델 목록이 자동으로 병합됩니다. 예를 들어 Azure OpenAI 연결 2개(미국/한국 리전)를 추가하면 양쪽 리전의 모델이 모두 표시됩니다. 이때 **Prefix ID**로 구분하세요.
</Tip>

## Ollama API 연결

로컬 또는 사내 서버의 Ollama 인스턴스와 연결합니다.

| 설정 항목 | 설명 | 예시 |
|-----------|------|------|
| **Base URL** | Ollama 서버 주소 | `http://localhost:11434` |

여러 서버를 추가하여 부하를 분산할 수 있습니다. Ollama는 외부 네트워크 없이 완전한 데이터 프라이버시를 보장합니다.

## 직접 연결 (Direct Connections)

관리자가 설정한 연결 외에, **사용자가 개인 API 키로 LLM에 직접 연결**하도록 허용하는 기능입니다.

| 설정 항목 | 설명 | 기본값 |
|-----------|------|:------:|
| **직접 연결 허용** | 사용자별 개인 API 연결 허용 | 활성화 |

활성화하면 사용자가 **개인 설정 > Connections**에서 OpenAI 호환 API를 추가할 수 있습니다.

<Note>
  직접 연결은 **OpenAI 호환 API만** 지원합니다 (Ollama 미지원). 사용자가 연결하는 API 서버에서 CORS 설정이 올바르게 구성되어 있어야 합니다.
</Note>

## Cloud Accounts

글로벌 클라우드 서비스 계정을 설정합니다. 여기서 설정한 Service Account Key는 개별 기능(TTS, STT, 이미지 등)에서 별도 키가 없을 때 fallback으로 사용됩니다.

| 제공자 | 설정 항목 |
|--------|-----------|
| **Google Cloud** | Service Account Key (JSON) |

<Note>
  Service Account Key를 비워두면 Application Default Credentials(ADC)를 사용합니다.
</Note>

## Image Attachment Mode

채팅에 첨부된 이미지의 처리 및 저장 방식을 설정합니다.

| 설정 항목 | 설명 | 기본값 |
|-----------|------|--------|
| **Image Upload Mode** | Base64 Inline 또는 Cloud Storage | `Base64 Inline` |
| **Storage Provider** | Local, AWS S3, Azure Blob, Google Cloud Storage | `Local` |

<Tabs>
  <Tab title="AWS S3">
    | 설정 항목 | 설명 |
    |-----------|------|
    | **Bucket Name** | S3 버킷 이름 |
    | **Region** | AWS 리전 |
    | **Endpoint URL** | 커스텀 Endpoint (선택) |
    | **Access Key ID** | 인증 키 ID |
    | **Secret Access Key** | 비밀 키 |
    | **Key Prefix** | 파일 경로 접두사 (선택) |
  </Tab>
  <Tab title="Azure Blob">
    | 설정 항목 | 설명 |
    |-----------|------|
    | **Storage Endpoint** | Azure Blob Storage Endpoint |
    | **Container Name** | 컨테이너 이름 |
    | **Storage Key** | 인증 키 |
  </Tab>
  <Tab title="Google Cloud Storage">
    | 설정 항목 | 설명 |
    |-----------|------|
    | **Bucket Name** | GCS 버킷 이름 |
    | **Authentication** | Global Google Cloud Key 또는 Custom Key 선택 |
  </Tab>
</Tabs>

## Shared Storage (파일 업로드)

문서, PDF 등 업로드 파일의 저장 위치를 설정합니다. Image Attachment Mode와 동일한 스토리지 제공자 옵션을 지원합니다.

| 설정 항목 | 설명 | 기본값 |
|-----------|------|--------|
| **File Storage Provider** | Local, AWS S3, Azure Blob, Google Cloud Storage | `Local` |

<Note>
  스토리지 설정 변경 후 **"Test"** 버튼으로 연결 상태를 반드시 확인하세요.
</Note>
