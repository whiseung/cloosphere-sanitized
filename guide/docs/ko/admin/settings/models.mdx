**관리자 > 설정 > 모델** 탭에서 설정합니다.

## 모델 목록

Workspace 모델(관리자가 직접 생성/커스텀한 모델)과 Base 모델(연결된 API에서 가져온 시스템 모델)을 합쳐서 표시합니다.

| 항목 | 설명 |
|------|------|
| **모델명** | 사용자에게 표시되는 모델 이름 |
| **ID** | API 호출 시 사용되는 모델 식별자 |
| **Enabled / Disabled** | 해당 모델의 사용 가능 여부 토글 (Switch) |

### 모델 숨김 (Shift+Click)

모델 목록에서 **Shift 키를 누른 상태**로 모델 행의 아이콘을 클릭하면 숨김/표시 토글이 나타납니다. 숨김 처리된 모델은 `meta.hidden` 상태로 관리되며, 사용자의 모델 선택 목록에서 보이지 않습니다.

<Note>
  숨김(Hidden)과 비활성화(Disabled)는 별개 기능입니다.
  숨김은 모델 선택 목록에서 감추는 것이고, 비활성화는 모델 자체를 사용 불가로 전환합니다.
</Note>

## 기본 모델 및 순서 설정

모델 목록 우측 상단의 설정(톱니바퀴) 아이콘을 클릭하면 모델 설정 모달이 열립니다.

| 설정 항목 | 설명 |
|-----------|------|
| **기본 모델** | 새 채팅 시작 시 자동 선택되는 모델 |
| **모델 순서** | 모델 선택 목록의 표시 순서 (드래그 또는 화살표로 조정) |

---

## 모델 활성화/비활성화

모델별 **Enabled / Disabled** 토글(Switch)로 필요한 모델만 사용자에게 노출할 수 있습니다.

<Note>
  비활성화(Disabled)된 모델은 사용자의 모델 선택 드롭다운에 표시되지 않습니다.
  에이전트의 base model로 지정된 모델은 비활성화해도 에이전트 실행에는 영향이 없습니다.
</Note>

## 모델 커스터마이징

모델 행의 **편집(연필) 아이콘**을 클릭하면 해당 모델을 커스터마이징할 수 있습니다.

| 설정 | 설명 |
|------|------|
| **이름/설명** | 사용자에게 표시되는 모델 이름과 설명 변경 |
| **시스템 프롬프트** | 해당 모델의 기본 시스템 프롬프트 설정 |
| **답변 포맷 프롬프트** | 응답 형식 지침 설정 |
| **파라미터** | Temperature, Top P, Top K, Stop 토큰 등 |
| **리소스 연결** | 지식기반, DB, 도구, 가드레일, 용어집 연결 |
| **기능 설정** | 웹 검색, 이미지 생성 활성화 |
| **자동 평가** | 응답 품질 자동 모니터링 |
| **접근 권한** | 특정 그룹/사용자에게만 노출 |

<Note>
  모델을 커스터마이징하면 해당 모델에 대한 **Workspace 설정 레코드**가 생성됩니다. 이것은 에이전트와 달리 기반 모델을 직접 수정하는 것이 아니라 **오버라이드 설정**을 추가하는 개념입니다.
</Note>

## 사용자에게 노출되는 모델 제어

| 메커니즘 | 효과 | 설정 위치 |
|---------|------|----------|
| **활성화/비활성화** (토글) | 비활성 → 모델 선택 목록에서 제외 | 모델 행 토글 |
| **숨기기** (Shift+클릭) | 숨김 → 모델 선택 목록에서 제외 | 모델 행 Shift+아이콘 또는 메뉴 |
| **기본 모델** | 새 채팅 시 자동 선택 | 설정 모달 |
| **표시 순서** | 모델 선택 드롭다운 순서 | 설정 모달 (드래그) |
| **접근 권한** | 특정 그룹/사용자만 접근 | 모델 편집 > 접근 권한 |

<Tip>
  **비활성화**와 **숨기기**는 사용자에게 같은 효과(목록에서 미노출)지만 관리자 화면에서 구분됩니다. 일시적으로 감추려면 숨기기를, 사용을 완전히 차단하려면 비활성화를 사용하세요.
</Tip>

## Ollama 모델 관리

모델 목록 상단의 **다운로드 아이콘**을 클릭하면 Ollama 모델 관리 모달이 열립니다.

| 기능 | 설명 |
|------|------|
| **Pull** | Ollama 모델 레지스트리에서 모델 다운로드 (최대 3개 동시) |
| **Update All** | 모든 Ollama 모델을 최신 버전으로 업데이트 |
| **Upload** | GGUF 파일 업로드 또는 URL에서 다운로드하여 모델 생성 |
| **Delete** | Ollama 모델 삭제 |

<Note>
  Ollama 인스턴스가 여러 개 연결되어 있으면 관리할 인스턴스를 먼저 선택해야 합니다.
</Note>

## 모델 프리셋 Import / Export

모델 설정을 JSON 형태로 백업하고 복원할 수 있습니다. 이 기능은 **관리자 전용**입니다.

| 기능 | 설명 |
|------|------|
| **Import Presets** | JSON 파일에서 모델 프리셋 복원 |
| **Export Presets** | 현재 모델 프리셋을 JSON 파일로 다운로드 |
| **단일 Export** | 모델 행의 더보기 메뉴(⋯) > Export |

<Warning>
  Import 시 기존 모델 설정이 덮어씌워집니다. 반드시 Export로 백업 후 진행하세요.
</Warning>

<AccordionGroup>
  <Accordion title="Import 시 어떤 모델이 가져와지나요?" icon="circle-question">
    `base_model_id`가 null인 모델(Workspace 커스텀 설정)만 Import 됩니다. 에이전트(`base_model_id`가 다른 모델을 참조)는 이 탭에서 Import되지 않습니다 — 에이전트는 워크스페이스 > 에이전트에서 별도로 Import하세요.
  </Accordion>

  <Accordion title="모든 모델 설정을 초기화하려면?" icon="circle-question">
    설정 모달(톱니바퀴) 하단의 **Reset All Models** 버튼을 클릭하면 모든 Workspace 커스텀 설정이 삭제됩니다. 연결된 기본 모델 목록은 유지되지만, 이름/설명/프롬프트 등 커스터마이징 내용이 모두 초기화됩니다.
  </Accordion>
</AccordionGroup>
