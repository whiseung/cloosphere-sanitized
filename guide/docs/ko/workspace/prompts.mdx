프롬프트는 자주 사용하는 질문 형식이나 지시사항을 **템플릿으로 저장**하고, `/` 명령어로 빠르게 호출할 수 있는 기능입니다.
검증된 프롬프트를 팀과 공유하면 일관된 품질의 AI 응답을 얻을 수 있습니다.

```mermaid
flowchart LR
    A[/ 명령어 입력] --> B[프롬프트 선택]
    B --> C[변수 값 입력]
    C --> D[템플릿 확장]
    D --> E[AI 처리]
    E --> F[응답 생성]
```

---

## 프롬프트란?

프롬프트는 AI에게 보내는 질문이나 지시사항의 **재사용 가능한 템플릿**입니다. 매번 긴 지시사항을 입력하는 대신, 명령어 한 줄로 호출할 수 있습니다.

### 왜 프롬프트를 사용하나요?

| 장점 | 설명 |
|------|------|
| **시간 절약** | 긴 지시사항을 매번 입력할 필요 없음 |
| **일관성** | 동일한 형식의 결과물을 반복 생성 |
| **팀 협업** | 효과적인 프롬프트를 팀원과 공유 |
| **품질 향상** | 검증된 프롬프트로 더 나은 응답 유도 |

## 프롬프트 생성

<Steps>
  <Step title="새 프롬프트 만들기">
    **워크스페이스 > 프롬프트**에서 우측 상단의 **+** 버튼을 클릭합니다.

    {/* SCREENSHOT: prompts-create
         화면: 프롬프트 생성 폼
         영역: 명령어 + 제목 + 내용 textarea
         상태: 예시 내용 입력된 상태
         하이라이트: 없음 */}
    <Frame caption="명령어, 제목, 프롬프트 내용을 입력합니다">
      <img src="/images/workspace/prompts-create-top.png" alt="프롬프트 생성 폼" />
    </Frame>

    | 필드 | 설명 | 예시 |
    |------|------|------|
    | **명령어** | `/`로 시작하는 단축 명령 | `email-draft` |
    | **제목** | 프롬프트 표시 이름 | "비즈니스 이메일 작성" |

    <Warning>
      명령어에는 **영문 소문자, 숫자, 하이픈(`-`)만** 사용할 수 있습니다. 한글, 공백, 특수문자는 허용되지 않습니다.
    </Warning>
  </Step>

  <Step title="프롬프트 내용 작성">
    프롬프트 템플릿을 작성합니다. Markdown 문법을 사용할 수 있습니다.

    ```markdown
    다음 내용으로 전문적인 비즈니스 이메일을 작성해주세요.

    ## 작성 규칙
    - 공손하고 전문적인 톤
    - 명확하고 간결한 문장
    - 적절한 인사말과 맺음말 포함

    ## 이메일 내용
    {{content}}
    ```
  </Step>

  <Step title="변수 설정 (선택)">
    `{{변수명}}` 구문을 사용하여 동적 값을 입력받을 수 있습니다.
    프롬프트 사용 시 변수 위치에 실제 값이 대입됩니다.

    ```markdown
    {{recipient}}님께 보내는 {{purpose}}에 관한 이메일을 작성해주세요.

    내용: {{content}}
    ```

    <Tip>
      변수명은 한글도 가능합니다. 예: `{{받는사람}}`, `{{주제}}`
    </Tip>

    #### 시스템 변수

    다음 내장 변수를 사용하면 자동으로 값이 채워집니다.

    | 변수 | 설명 |
    |------|------|
    | `{{CLIPBOARD}}` | 클립보드 내용 |
    | `{{USER_NAME}}` | 현재 사용자 이름 |
    | `{{USER_LANGUAGE}}` | 사용자 언어 설정 |
    | `{{USER_LOCATION}}` | 사용자 위치 |
    | `{{CURRENT_DATE}}` | 오늘 날짜 |
    | `{{CURRENT_TIME}}` | 현재 시간 |
    | `{{CURRENT_DATETIME}}` | 현재 날짜와 시간 |
    | `{{CURRENT_TIMEZONE}}` | 현재 시간대 |
    | `{{CURRENT_WEEKDAY}}` | 현재 요일 |
  </Step>

  <Step title="접근 권한 설정">
    프롬프트를 공개할 범위를 선택합니다.

    | 옵션 | 설명 |
    |------|------|
    | **공개 (Public)** | 모든 사용자가 `/` 명령어로 사용 가능 |
    | **비공개 (Private)** | 선택한 그룹 또는 조직 단위의 구성원만 사용 가능. 그룹/조직을 지정하지 않으면 생성자만 접근 |

    <Note>
      공개 옵션은 관리자 또는 프롬프트 공개 권한(`sharing.public_prompts`)이 부여된 사용자만 선택할 수 있습니다.
    </Note>
  </Step>

  <Step title="저장">
    **저장 및 생성** 버튼을 클릭합니다.
  </Step>
</Steps>

## 프롬프트 사용

### `/` 명령어로 호출

채팅창에 `/`를 입력하면 사용 가능한 프롬프트 목록이 자동완성으로 나타납니다.

```
/email-draft 클라이언트에게 미팅 일정 조율 요청
```

프롬프트에 `{{content}}` 변수가 정의되어 있으면, 명령어 뒤에 입력한 텍스트가 해당 변수에 대입됩니다.

<Tip>
  여러 변수가 있는 프롬프트를 사용할 때, 각 변수 값을 입력한 후 **Tab** 키를 누르면 다음 변수 입력란으로 이동합니다.
</Tip>

## 프롬프트 예시

<Tabs>
  <Tab title="이메일 작성">
    **명령어:** `/email-draft`

    ```markdown
    다음 정보를 바탕으로 전문적인 비즈니스 이메일을 작성해주세요.

    ## 요구사항
    - 공손하고 전문적인 톤 유지
    - 핵심 내용을 명확히 전달
    - 적절한 인사말과 맺음말 포함
    - 필요시 다음 단계(Action Item) 명시

    ## 이메일 내용
    {{content}}
    ```

    **사용 예시:**
    ```
    /email-draft 클라이언트에게 다음 주 미팅 일정을 화요일 2시로 변경 요청
    ```
  </Tab>

  <Tab title="보고서 작성">
    **명령어:** `/report`

    ```markdown
    다음 주제로 체계적인 보고서를 작성해주세요.

    ## 보고서 구조
    1. 요약 (Executive Summary)
    2. 현황 분석
    3. 문제점 및 원인
    4. 개선 방안
    5. 실행 계획
    6. 기대 효과

    ## 주제
    {{topic}}
    ```
  </Tab>

  <Tab title="코드 리뷰">
    **명령어:** `/code-review`

    ```markdown
    다음 코드를 리뷰하고 개선점을 제안해주세요.

    ## 리뷰 관점
    1. **가독성**: 코드가 이해하기 쉬운가?
    2. **효율성**: 성능 개선 여지가 있는가?
    3. **보안**: 보안 취약점은 없는가?
    4. **베스트 프랙티스**: 관례를 따르고 있는가?
    5. **에러 처리**: 예외 상황을 적절히 처리하는가?

    ## 코드
    {{code}}
    ```
  </Tab>

  <Tab title="회의록 정리">
    **명령어:** `/meeting-notes`

    ```markdown
    다음 회의 내용을 정리해주세요.

    ## 회의록 형식
    ### 회의 정보
    - 일시:
    - 참석자:
    - 안건:

    ### 논의 내용
    ### 결정 사항
    ### Action Items
    | 담당자 | 업무 | 기한 |
    |--------|------|------|

    ## 회의 내용
    {{content}}
    ```
  </Tab>

  <Tab title="번역">
    **명령어:** `/translate`

    ```markdown
    다음 텍스트를 {{target_lang}}로 번역해주세요.

    ## 번역 규칙
    - 원문의 의미와 뉘앙스 유지
    - 자연스러운 표현 사용
    - 전문 용어는 적절히 처리
    - 문화적 맥락 고려

    ## 원문
    {{text}}
    ```
  </Tab>
</Tabs>

## 프롬프트 관리

### 편집 및 복제

- **편집**: 프롬프트 목록에서 **연필 아이콘**을 클릭하여 편집 페이지로 이동합니다
- **복제**: 프롬프트 메뉴(⋮)에서 **복제**를 선택하여 새 버전을 만듭니다

<Note>
  쓰기 권한이 있거나 관리자인 경우 다른 사용자의 프롬프트도 편집할 수 있습니다. 권한이 없는 경우 **복제**하여 자신만의 버전을 만든 후 수정하세요.
</Note>

### 내보내기 / 가져오기

프롬프트를 JSON 파일로 내보내고 가져올 수 있습니다.

- 팀원과 프롬프트 공유
- 백업 및 복원
- 다른 환경으로 이동

<Note>
  일괄 내보내기/가져오기는 **관리자만** 사용할 수 있습니다. 개별 프롬프트 내보내기는 프롬프트 메뉴(⋮)에서 모든 사용자가 사용 가능합니다.
</Note>

### 삭제

더 이상 필요 없는 프롬프트는 삭제할 수 있습니다. 생성자, 쓰기 권한이 있는 사용자, 또는 관리자가 삭제할 수 있습니다.

## 효과적인 프롬프트 작성 팁

<Accordion title="1. 역할 정의">
  AI에게 명확한 역할을 부여하면 더 전문적인 답변을 얻을 수 있습니다.

  ```markdown
  당신은 10년 경력의 마케팅 전문가입니다.
  ```
</Accordion>

<Accordion title="2. 구체적인 지시">
  원하는 결과물을 구체적으로 설명하세요. 분량, 톤, 포함할 내용을 명시합니다.

  ```markdown
  ## 요구사항
  - 500자 내외로 작성
  - 전문적이지만 친근한 톤
  - 3개의 핵심 메시지 포함
  ```
</Accordion>

<Accordion title="3. 출력 형식 지정">
  원하는 형식(표, 목록, 마크다운 등)을 명시하면 일관된 결과를 얻을 수 있습니다.

  ```markdown
  ## 출력 형식
  | 항목 | 내용 |
  |------|------|
  | 제목 | ... |
  | 본문 | ... |
  ```
</Accordion>

<Accordion title="4. 예시 제공">
  원하는 결과의 예시를 보여주면 AI가 형식과 스타일을 정확히 파악합니다.

  ```markdown
  ## 예시
  입력: "프로젝트 미팅"
  출력: "프로젝트 A 2차 진행 현황 공유 미팅"
  ```
</Accordion>

<Accordion title="5. 제약 조건 명시">
  피해야 할 것을 알려주면 원치 않는 결과를 방지할 수 있습니다.

  ```markdown
  ## 제한사항
  - 전문 용어 최소화
  - 비속어 사용 금지
  - 개인정보 포함 금지
  ```
</Accordion>

## 팀 프롬프트 공유

### 접근 권한

프롬프트 생성 시 접근 권한을 설정하여 공유 범위를 지정합니다.

| 옵션 | 설명 |
|------|------|
| **공개 (Public)** | 모든 사용자가 `/` 명령어로 사용 가능 |
| **비공개 (Private)** | 선택한 그룹/조직 단위의 구성원만 사용 가능. 미지정 시 생성자만 접근 |

## FAQ

<AccordionGroup>
  <Accordion title="프롬프트 명령어에 한글을 사용할 수 있나요?" icon="circle-question">
    아니요. 명령어에는 **영문 소문자, 숫자, 하이픈(`-`)만** 사용할 수 있습니다. 예: `/email-draft`, `/code-review`, `/meeting-notes`
  </Accordion>

  <Accordion title="프롬프트에 파일을 포함할 수 있나요?" icon="circle-question">
    프롬프트 템플릿 자체에는 파일을 포함할 수 없지만, 프롬프트 사용 시 파일을 함께 첨부할 수 있습니다.
  </Accordion>

  <Accordion title="다른 사람의 프롬프트를 수정할 수 있나요?" icon="circle-question">
    쓰기 권한이 부여된 경우 또는 관리자만 수정할 수 있습니다. 권한이 없으면 **복제**하여 자신만의 버전을 만든 후 수정하세요.
  </Accordion>

  <Accordion title="변수에 긴 텍스트를 넣을 수 있나요?" icon="circle-question">
    네, 변수에 입력할 수 있는 텍스트 길이에 제한은 없습니다. 문서 전체 내용도 변수로 전달할 수 있습니다.
  </Accordion>
</AccordionGroup>
