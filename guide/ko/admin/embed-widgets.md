# 임베드 위젯

Cloosphere의 AI 채팅을 외부 웹사이트에 손쉽게 임베드할 수 있는 위젯 기능입니다. 한 줄의 스크립트 태그만으로 그룹웨어, 사내 포털, 마케팅 사이트 등 어디든 AI 비서를 심을 수 있습니다.

## 개요

임베드 위젯은 다음과 같은 시나리오에 활용됩니다:

- **사내 그룹웨어/포털 통합**: 직원이 휴가 신청, 품의서 작성 등의 폼을 작성할 때 AI가 도와줌
- **고객 지원 챗봇**: 회사 홈페이지에 24/7 AI 상담원 추가
- **사내 위키/문서 사이트**: 문서를 보면서 AI에게 즉석에서 질문
- **데모/마케팅 페이지**: AI 솔루션 시연용

## 위젯 만들기

### 1. 위젯 생성

**관리자 → 설정 → 임베드 위젯** 메뉴에서 **위젯 생성** 버튼을 클릭합니다.

기본 정보를 입력합니다:

| 항목 | 설명 |
|------|------|
| **위젯 이름** | 관리자 화면에서 표시되는 이름 |
| **설명** | 위젯의 용도 메모 (선택) |
| **모델** | 위젯에 사용할 기본 에이전트 또는 모델을 선택 |
| **추가 에이전트 (/ 로 전환)** | 사용자가 채팅 입력창에서 `/` 를 눌러 전환할 수 있는 추가 에이전트 목록 (선택) |
| **시스템 프롬프트** | 에이전트의 역할/제약을 정의 (선택, 에이전트 자체 프롬프트 외 추가 지시) |
| **환영 메시지** | 채팅창을 처음 열었을 때 표시되는 메시지 |
| **허용 도메인** | 위젯이 동작할 수 있는 도메인 목록 (콤마 구분, 비워두면 모든 도메인 허용) |
| **활성화** | 위젯 사용 여부 |

저장하면 위젯 ID와 함께 임베드 코드 스니펫이 생성됩니다.

#### 슬래시 커맨드(Slash Command)로 에이전트 전환

위젯 편집 화면의 **추가 에이전트** 체크박스에서 사용자가 선택할 수 있는 보조 에이전트들을 등록할 수 있습니다. 등록된 위젯에서 사용자가 입력창 첫 글자에 `/` 를 입력하면 등록된 에이전트 목록이 피커로 표시되고, 방향키 / Tab / Enter / Esc 로 전환할 수 있습니다. 현재 활성 에이전트는 입력창 상단에 인디케이터로 표시됩니다.

### 2. 임베드 코드 복사

위젯 목록에서 **`</>` 아이콘**을 클릭하면 다음과 같은 임베드 코드가 표시됩니다:

```html
<script
  src="https://your-cloosphere.com/static/embed/embed.js"
  data-widget-id="abc12345-..."
></script>
```

이 한 줄을 외부 사이트의 HTML `<body>` 안에 추가하면 우측 하단에 채팅 버블이 나타납니다.

### 3. 디자인 커스터마이징

위젯 목록에서 **디자인 아이콘**을 클릭하면 디자인 편집 모달이 열립니다. 좌측에서 설정하고 우측에서 라이브 미리보기를 확인할 수 있습니다.

#### Layout 탭

| 설정 | 설명 |
|------|------|
| **표시 모드** | 버블 / 좌측 사이드 / 우측 사이드 / 하단 사이드 / 인라인 / 전체 화면 |
| **테마** | 자동 / 라이트 / 다크 |
| **위치** | 우측 하단 / 좌측 하단 (버블 모드 전용) |
| **버블 열림 스타일** | 팝업 / 우측 사이드 / 좌측 사이드 / 하단 사이드 (버블 모드 전용) |
| **드래그 가능한 버블** | 사용자가 버블 버튼을 드래그해서 옮길 수 있게 (위치는 브라우저별 저장) |
| **리사이즈 허용** | 사용자가 위젯 모서리를 드래그해 크기를 직접 조절할 수 있게 함. 사이드 모드에서는 위젯이 커지면 호스트 페이지 본문이 자동으로 밀려 가려지지 않도록 조정됩니다 |
| **버블 크기** | 40~96px 슬라이더 |
| **너비/높이** | 위젯 크기 직접 지정 (예: `400px`, `100%`, `80vh`) |

**버블 열림 스타일 상세:**

- **팝업 (Popup)**: 기본값. 버블 위에 작은 패널이 떠 있고, 버블이 X 아이콘으로 변하며 다시 클릭하면 닫힘
- **사이드 (Side Right/Left/Bottom)**: 버블 클릭 시 화면 옆 또는 하단에 풀 사이즈 패널로 뜨고 버블은 숨김. 헤더의 X 버튼으로만 닫을 수 있음. 더 넓은 작업 공간이 필요한 경우 권장

**6가지 표시 모드:**

- **버블 (Bubble)**: 우측 하단 플로팅 버튼, 클릭하면 패널이 열림 (Intercom, Zendesk 스타일)
- **사이드 패널 (Side Panel)**: 화면 좌측 또는 우측에 고정된 패널
- **하단 사이드 패널 (Side Bottom)**: 화면 하단에 고정된 패널. 긴 페이지에서 항상 보이는 보조 도우미로 유용
- **인라인 (Inline)**: 페이지 특정 영역에 직접 삽입
- **전체 화면 (Fullscreen)**: 화면 전체를 채팅으로 사용

#### Icons 탭

- **버블 아이콘**: 채팅, 스파클, 메시지, 로봇, 헤드셋, 도움말, 번개 등 7가지 프리셋 + 직접 업로드
- **전송 버튼 아이콘**: 종이비행기, 화살표, 별, 로켓 등 7가지 프리셋 + 직접 업로드

> 프리셋 아이콘(SVG)은 색상 변경이 가능하고, 업로드한 이미지는 원본 색상이 유지됩니다.

#### Header 탭

- **헤더 표시 여부**: 헤더 영역을 숨기거나 보이게 설정
- **헤더에 닫기 버튼 표시**: 헤더 우측에 X 버튼 표시 여부 (기본값 ON). 사이드 모드에서는 위젯을 닫는 유일한 방법이므로 사이드 모드 사용 시 권장
- **헤더 텍스트**: 위젯 상단에 표시되는 제목 (기본값: 위젯 이름)
- **헤더 배경 / 헤더 텍스트 색상**

#### Colors 탭

**테마 프리셋** (Colors 탭 상단):

12가지 미리 설계된 색상 조합 중 하나를 클릭하면 모든 색상과 라이트/다크 모드가 한 번에 적용됩니다:

| 프리셋 | 분위기 | 라이트/다크 |
|---|---|---|
| **Light** / **Dark** | 프레임워크 디폴트 (커스텀 색상 없이 기본 스타일) | 각각 |
| **Slate** | 미니멀 흑백 | Light |
| **Midnight** | 슬레이트 + 인디고, 차분한 다크 | Dark |
| **Ocean** | 코퍼레이트 블루 | Light |
| **Aurora** | 바이올렛 + 핑크 | Light |
| **Sunset** | 따뜻한 오렌지 | Light |
| **Forest** | 자연스러운 그린 | Light |
| **Rose** | 우아한 로즈 | Light |
| **Cyberpunk** | 다크 + 네온 사이안 | Dark |
| **Ember** | 따뜻한 다크 + 앰버 글로우 | Dark |
| **Mono** | 순수 흑백 | Light |

프리셋 적용 후에도 아래 개별 색상 입력에서 미세 조정 가능합니다.

**개별 색상 설정**:

- **버블 배경 / 버블 아이콘 색상**: 플로팅 버튼의 색상
- **배경 색상**: 채팅창 전체 배경
- **메시지 텍스트 색상**: 사용자/봇 메시지 공통 텍스트 색상
- **전송 버튼 배경 / 아이콘 색상**

#### Chat 탭

- **환영 메시지**: 채팅창을 처음 열었을 때 표시
- **세션당 최대 메시지 수**: 메시지 수 제한 (0 = 무제한)
- **봇 아이콘 표시 여부**: 응답 메시지 옆 에이전트 아이콘 표시 토글

#### Features 탭

- **파일 업로드**: 사용자가 위젯에서 파일을 첨부할 수 있는지
- **마크다운**: 마크다운 렌더링 활성화
- **코드 하이라이팅**: 코드 블록 구문 강조
- **웹 검색**: 위젯에서 웹 검색 도구 사용 가능 여부

#### SSO 탭

호스트 사이트가 자체 SSO(Microsoft Entra ID, Google, GitHub, OIDC 호환 IdP 등)로 사용자를 인증하고, 그 토큰을 받아 Cloosphere 사용자로 자동 매핑/로그인 할 수 있게 합니다. 호스트 사용자가 별도로 위젯에서 다시 로그인할 필요가 없어집니다.

| 설정 | 설명 |
|---|---|
| **SSO 토큰 교환 활성화** | 위젯이 외부 SSO 토큰을 받아 Cloosphere JWT로 교환할 수 있게 함 |
| **알 수 없는 사용자 자동 가입** | 토큰의 이메일이 Cloosphere에 없는 사용자라면 자동으로 계정 생성 |
| **신규 사용자 기본 역할** | 자동 가입된 사용자의 초기 역할 (User / Pending / Admin) |
| **허용 공급자** | 호스트 사이트가 사용할 수 있는 SSO 공급자 화이트리스트 (Microsoft / Google / GitHub / OIDC) |

**공급자별 추가 설정:**

- **Microsoft Entra ID**: `Tenant ID` (특정 테넌트 GUID 또는 `common`), `Trusted Audiences` (호스트 사이트의 client_id 화이트리스트, 비우면 해당 테넌트의 모든 앱 토큰 허용)
- **Google**: `Trusted Audiences` (호스트 사이트의 Google client_id, 선택)
- **GitHub**: 추가 옵션 없음
- **Generic OIDC**: `Trusted Issuers` (Keycloak/Auth0/Okta 등의 issuer URL, 필수), `Trusted Audiences` (선택)

> **호스트 사이트 개발자가 이 위젯을 자기 사이트에 통합하려면**, [임베드 위젯 호스트 통합 가이드](embed-widgets-integration.md)를 참고하세요. SSO 토큰 전달, Soft Navigation, 트러블슈팅 등 통합에 필요한 모든 내용을 다룹니다.

#### Guest 탭

로그인하지 않은 외부 방문자도 위젯을 사용할 수 있는 **게스트 모드**를 설정합니다. 공개 웹사이트나 마케팅 페이지에 챗봇을 배치할 때 유용합니다.

| 설정 | 설명 |
|---|---|
| **게스트 허용** | 미로그인 방문자의 위젯 사용 여부 |
| **사용자 정보 수집** | 대화 시작 전에 이름/이메일 등 기본 정보를 받을지 여부 |
| **필수 항목** | 반드시 입력해야 하는 필드 (예: 이름) |
| **선택 항목** | 입력해도 되고 안 해도 되는 필드 (예: 이메일) |
| **자동 진행** | 입력 모달 없이 자동으로 익명 게스트 세션 생성 |
| **세션 만료** | 게스트 토큰의 유효 기간 (예: `24h`) |

> **참고:** 게스트 모드와 SSO 모드는 상호 배타적입니다. 한 위젯에서는 로그인 / 게스트 / SSO 중 하나의 인증 흐름만 사용할 수 있습니다.

> **디자인 옵션 확장:** 색상·아이콘·버튼 프리셋을 비롯한 세밀한 디자인 옵션이 지속적으로 추가되고 있어, 호스트 사이트 브랜드 톤에 더 가깝게 위젯을 맞출 수 있습니다.

## 사용자 인증

임베드 위젯은 다섯 가지 인증 방식을 지원합니다 (우선순위 순):

### 1. SSO 토큰 교환 (권장 — 호스트 사이트 SSO 통합)

호스트 사이트가 자체 SSO(Microsoft Entra ID, Google, OIDC 등)로 사용자를 인증한 뒤, 그 id_token을 위젯에 넘기면 위젯이 백엔드에서 토큰을 검증·매핑·교환해 자동 로그인합니다. 사용자는 위젯에서 별도로 로그인할 필요가 없습니다.

이 방식을 사용하려면 **SSO 탭**에서 활성화 + 공급자 등록 후, 호스트 사이트 개발자에게 [호스트 통합 가이드](embed-widgets-integration.md)를 안내하세요.

### 2. 외부 서비스가 Cloosphere JWT 직접 전달

호스트 사이트가 백엔드에서 이미 Cloosphere JWT를 발급받은 경우, `data-token` 속성으로 직접 전달:

```html
<script
  src="https://your-cloosphere.com/static/embed/embed.js"
  data-widget-id="..."
  data-token="JWT_TOKEN_HERE"
></script>
```

이 방식은 호스트가 Cloosphere의 사용자 매핑/JWT 발급을 직접 수행할 수 있는 경우에 적합합니다. 일반적인 SSO 통합이라면 1번 방식이 더 간단합니다.

### 3. 게스트 모드

위젯의 **Guest 탭**에서 게스트 모드가 활성화되어 있으면 로그인 없이 바로 대화를 시작할 수 있습니다.

- 위젯을 처음 여는 방문자에게 이름/이메일 같은 기본 정보를 입력받거나(선택), 자동 진행 설정이 켜져 있으면 곧바로 익명 세션을 시작
- 서버는 짧은 만료 시간의 게스트 토큰을 발급하고, 해당 세션 범위에서만 대화가 유지됨
- 공개 웹사이트/마케팅 페이지에 배치하는 24/7 상담 챗봇에 적합

### 4. 위젯 자체 로그인 (자동)

위 세 방식이 모두 안 될 때, 위젯이 처음 열리면 **로그인 화면**이 자동으로 표시됩니다:

- **OAuth 버튼**: Cloosphere에 OAuth 공급자(Microsoft, Google, GitHub 등)가 설정되어 있으면 자동으로 버튼이 표시됩니다. 클릭 시 팝업 윈도우로 OAuth 로그인을 진행합니다.
- **이메일/비밀번호 폼**: 일반 로그인 폼으로 Cloosphere 계정에 로그인할 수 있습니다.

로그인된 토큰은 브라우저의 sessionStorage에 위젯 단위로 저장되어, 같은 세션 내에서는 재로그인 없이 사용할 수 있습니다.

### 5. 세션 재사용

이전에 위젯에서 로그인한 적이 있다면, 같은 브라우저 세션에서 자동으로 토큰을 재사용합니다.

## 보안

임베드 위젯은 운영 환경에서 안전하게 사용할 수 있도록 다음 보안 메커니즘을 제공합니다:

### 허용 도메인 제한

위젯 생성/편집 시 **허용 도메인** 필드에 사용 가능한 도메인을 지정할 수 있습니다:

```
*.example.com, app.mycompany.com
```

설정된 도메인 외에서 위젯을 로드하면 거부됩니다. 와일드카드(`*.example.com`)를 지원합니다.

### 페이지 조작 보안

임베드 위젯의 AI가 외부 페이지의 폼이나 버튼을 직접 조작할 수 있는 기능을 사용할 때:

- 도메인 검증을 통해 의도하지 않은 사이트에서의 조작을 방지합니다
- 위젯별 허용 셀렉터 화이트리스트를 통해 조작 가능한 영역을 제한할 수 있습니다
- 모든 작업은 사용자의 브라우저에서 실행되며, 사용자가 직접 보고 검증할 수 있습니다

### CORS 정책

위젯 설정 조회 및 **SSO 토큰 교환 엔드포인트**는 외부 사이트에서 호출할 수 있도록 CORS가 허용되어 있으나, **위젯 ID · 허용 도메인 · 공급자/Audience 화이트리스트**로 보호됩니다. 허용되지 않은 origin이나 등록되지 않은 공급자 토큰은 거부됩니다.

### Public/Admin API 분리

임베드 위젯 관련 백엔드 엔드포인트는 두 그룹으로 분리되어 있습니다.

| 구분 | prefix | 용도 |
|------|--------|------|
| **Public** | `/api/embed/v1/*` (또는 레거시 `/api/v1/embed-widgets/*`) | 외부 호스트 사이트가 호출하는 config / auth / sso / guest 엔드포인트. CORS 허용 |
| **Admin** | `/api/v1/embed-widgets/*` (CRUD) | 관리자만 사용하는 위젯 생성·수정·삭제 |

레거시 prefix 는 기존에 빌드된 호스트 사이트와의 호환성을 위해 함께 마운트되어 있습니다. 신규 통합 시에는 새 prefix(`/api/embed/v1/*`) 사용을 권장합니다. 자세한 내용은 [임베드 위젯 호스트 통합 가이드](embed-widgets-integration.md) 를 참고하세요.

## 외부 사이트와의 통합

### 인터랙티브 페이지 조작

임베드 위젯의 가장 강력한 기능은 AI가 사용자가 보고 있는 페이지를 직접 조작할 수 있다는 점입니다. 예를 들어:

- 사용자가 휴가 신청 페이지에서 "내일부터 3일 연차 신청해줘"라고 말하면
- AI가 폼 필드 구조를 파악하고 자동으로 값을 채움
- 사용자에게 검토를 요청한 후 제출 버튼 클릭

이 기능은 다음 도구를 통해 동작합니다:

| 도구 | 설명 |
|------|------|
| `get_page_info` | 사용자가 보고 있는 페이지의 URL과 폼 정보 조회 |
| `fill_form_field` | 단일 폼 필드에 값 입력 |
| `fill_form` | 여러 필드를 한 번에 입력 |
| `click_element` | 버튼/링크 클릭 |
| `read_form` | 폼의 현재 값 읽기 |
| `highlight_element` | 요소 강조 표시 |
| `navigate_to` | 페이지 이동 |

이 도구들은 에이전트를 위한 시스템 프롬프트에서 활용 방법을 안내해야 합니다.

### MCP 통합

외부 사이트의 백엔드 API와 통합하려면 **도구 연결(MCP)** 을 활용하세요. 그룹웨어 시스템의 폼 스키마, 사용자 정보, 비즈니스 데이터 등을 MCP 서버로 노출하면 임베드 위젯의 AI가 이를 활용해 더 정확한 작업을 수행할 수 있습니다.

자세한 내용은 [도구 연결](../workspace/tools.md) 가이드를 참조하세요.

### Soft Navigation 지원

위젯이 페이지 이동(`navigate_to`)을 호출할 때, 외부 사이트가 SPA 라우터를 사용한다면 페이지 새로고침 없이 부드럽게 이동할 수 있습니다. 외부 사이트에 다음과 같은 리스너를 추가하세요:

```javascript
window.addEventListener('cloosphere:navigate', (e) => {
  const { url } = e.detail;
  router.push(url); // Next.js 예시
  window.postMessage({ type: 'cloosphere:navigation-handled' }, '*');
});
```

리스너가 없으면 일반 페이지 이동으로 자동 폴백됩니다.

- **환영 메시지 활용**: 사용자가 어떤 작업을 요청할 수 있는지 환영 메시지로 안내하세요

### 에이전트 설정

- 페이지 조작 기능을 사용한다면, 시스템 프롬프트에 어떤 작업을 처리할 수 있는지 명시하세요
- 민감한 작업(제출, 삭제 등)은 항상 사용자 확인 후 실행하도록 안내하세요
- `get_page_info`를 먼저 호출해서 현재 페이지 컨텍스트를 파악하도록 워크플로우를 정의하세요

- **버전 관리**: 위젯 디자인 변경 시 이전 사용자의 캐시 문제를 고려하세요

## FAQ

**Q. 한 페이지에 여러 개의 위젯을 띄울 수 있나요?**
A. 권장하지 않습니다. 한 페이지에 한 위젯만 띄우는 것이 안정적입니다. 페이지마다 다른 위젯이 필요하다면, 페이지별로 다른 임베드 코드를 사용하세요.

**Q. 위젯 디자인을 변경하면 즉시 반영되나요?**
A. 네. 사용자가 페이지를 새로고침하면 새 디자인이 적용됩니다.

**Q. 위젯에서 발생한 채팅 기록은 어디에 저장되나요?**
A. Cloosphere의 일반 채팅과 동일하게 사용자 계정 단위로 저장됩니다. 관리자는 모니터링에서 확인할 수 있습니다.

**Q. 위젯 사용자가 제 Cloosphere 계정으로 로그인할 수 있나요?**
A. 네. 위젯이 자체 로그인 화면을 제공하므로, 외부 사이트에서 별도 통합 작업 없이도 사용자가 자신의 Cloosphere 계정으로 로그인할 수 있습니다.

**Q. 임베드 위젯에서 백그라운드 작업(채팅 제목 자동 생성 등)이 동작하나요?**
A. 임베드 위젯에서는 본 사이트의 알림이 노출되지 않도록, 채팅 제목/태그 자동 생성 등 일부 백그라운드 작업이 비활성화되어 있습니다.

**Q. 디자인 편집 모달에서 저장했더니 게스트/SSO/표시 모드 설정이 사라졌다가 다시 나타납니다. 왜죠?**
A. 이전 버전에서 디자인 모달 저장이 인증 탭(Guest/SSO)의 설정을 덮어쓰던 이슈가 있었는데, 현재는 디자인 모달에서 노출되지 않는 인증/표시 모드 설정이 그대로 보존되도록 수정되었습니다. 최신 버전에서는 해당 문제가 발생하지 않습니다.

**Q. 스트리밍 응답 중 타이핑 애니메이션이 더 매끄러워졌나요?**
A. 네. 응답 스트리밍 렌더링이 개선되어 긴 답변에서도 문장이 부드럽게 이어서 표시됩니다.
