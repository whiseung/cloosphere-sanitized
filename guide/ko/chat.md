# 채팅 기능

> Cloosphere의 채팅은 단순한 대화를 넘어, 파일 분석, 코드 실행, 웹 검색, 음성 대화까지 지원하는 종합 AI 작업 공간입니다.

```mermaid
flowchart TB
    subgraph 입력
        A[텍스트 입력] --> E[메시지 전송]
        B[파일 첨부] --> E
        C[음성 입력] --> E
        D[@ 명령어] --> E
    end

    subgraph AI_처리
        E --> F{웹 검색 필요?}
        F -->|Yes| G[웹 검색]
        F -->|No| H[LLM 처리]
        G --> H
        H --> I{지식베이스?}
        I -->|Yes| J[문서 검색]
        I -->|No| K[응답 생성]
        J --> K
    end

    subgraph 출력
        K --> L[응답 표시]
        L --> M[인용 출처]
        L --> N[코드 실행]
        L --> O[TTS 재생]
    end
```

## 모델 선택 및 관리

### 모델 선택기

모델 선택 드롭다운은 **유형별 분류**와 **정렬** 기능을 제공합니다.

<!-- 스크린샷: 모델 선택기 유형 분류 UI
     파일명: images/chat-model-selector-types.png
-->

| 유형 | 설명 |
|------|------|
| **에이전트** | 사전 구성된 AI 에이전트 (도구, 지식베이스 연동) |
| **모델** | 기본 LLM 모델 (GPT-4o, Claude 등) |

**정렬 옵션:**
- 이름순 (가나다/알파벳)
- 최근 사용순
- 즐겨찾기 우선

> 💡 **Tip**: 에이전트와 모델이 한 목록에 섞여 있을 때 유형 필터를 사용하면 원하는 항목을 빠르게 찾을 수 있습니다.

### 기본 모델 설정

자주 사용하는 모델을 기본값으로 설정할 수 있습니다.

1. 모델 선택 드롭다운 열기
2. 원하는 모델 선택
3. **"기본값으로 설정"** 클릭
![](../images/0131135449486.png)

### 멀티 모델 대화

하나의 질문에 여러 모델의 답변을 동시에 받아볼 수 있습니다.

1. 모델 선택에서 **"+"** 버튼 클릭
2. 추가 모델 선택
3. 질문 전송

![](../images/0131140227802.png)

**활용 사례:**
- 여러 모델의 답변 비교
- 중요한 결정 전 교차 검증
- 최적의 응답 선택 후 병합

## 메시지 입력

### 텍스트 입력

입력창에 질문이나 요청을 자유롭게 입력합니다.

**서식 지원:**
- **마크다운**: `**굵게**`, `*기울임*`, `` `코드` ``
- **줄바꿈**: `Shift + Enter`
- **전송**: `Enter` 또는 전송 버튼

### 파일 첨부

다양한 방법으로 파일을 첨부할 수 있습니다.

| 방법 | 설명 |
|------|------|
| **드래그 앤 드롭** | 파일을 채팅창으로 끌어다 놓기 |
| **파일 선택** | + 버튼 → 파일 업로드 |
| **이미지 캡처** | + 버튼 → 화면 캡처 |
| **URL 입력** | + 버튼 → URL로 콘텐츠 가져오기 |
| **클라우드 연동** | Google Drive, OneDrive, SharePoint |

**지원 파일 형식:**

| 카테고리 | 형식 |
|----------|------|
| 문서 | PDF, DOCX, PPTX, TXT, MD, HTML |
| 스프레드시트 | XLSX, CSV |
| 이미지 | PNG, JPG, GIF, WebP |
| 코드 | PY, JS, TS, Java, C++, 등 |
| 데이터 | JSON, XML, YAML |

### 이미지 분석

이미지를 첨부하면 AI가 내용을 분석하고 설명합니다.
![](../images/0131140435081.png)

**활용 예시:**
- 차트/그래프 데이터 추출
- 문서 이미지 OCR
- UI 디자인 피드백
- 오류 화면 분석

### 음성 입력

마이크 버튼을 클릭하여 음성으로 입력할 수 있습니다.

![](../images/0131140523235.png)

1. 🎤 **마이크 버튼** 클릭
2. 음성으로 말하기
3. 자동으로 텍스트 변환
4. 필요시 수정 후 전송

## 명령어 시스템

`@` 기호로 시작하는 명령어를 사용하여 빠르게 기능을 호출할 수 있습니다.

### @모델명 - 모델 지정

특정 모델에게 직접 질문할 수 있습니다.

```
@gpt-4o 이 코드를 리뷰해줘
```

### /프롬프트 - 템플릿 사용

저장된 프롬프트 템플릿을 불러옵니다.

```
/이메일작성 클라이언트에게 프로젝트 지연 안내 메일
```

### #지식베이스 - 문서 참조

특정 지식베이스의 문서를 참조하여 답변합니다.

```
#인사규정 연차 신청 절차가 어떻게 되나요?
```

## AI 응답 활용

### 응답 도구 모음

AI 응답 하단에는 다양한 도구가 제공됩니다.

![](../images/0131140738233.png)

| 버튼 | 기능 |
|------|------|
| ✏️ **편집** | 응답 수정 |
| 📋 **복사** | 전체 응답 복사 |
| 🔊 **음성** | TTS로 응답 읽기 |
| ℹ️ **정보** | 사용된 토큰량 확인 |
| 👍👎 **평가** | 응답 품질 피드백 |
| ▶️ **계속** | 응답 이어서 생성 |
| 🔄 **재생성** | 새로운 응답 요청 |

### 코드 실행

AI가 생성한 Python 코드를 바로 실행하고 결과를 확인할 수 있습니다.

**지원 기능:**
- Python 코드 실행
- 데이터 시각화 (matplotlib, plotly)
- 파일 생성 및 다운로드
- 실행 결과 확인

### 아티팩트 뷰어

AI가 생성한 HTML, CSS, JavaScript를 미리보기할 수 있습니다.

1. AI에게 웹 페이지나 컴포넌트 생성 요청
2. 응답에서 **"미리보기"** 버튼 클릭
3. 결과물 확인 및 코드 복사

### 인용 및 출처

지식베이스나 웹 검색을 참조한 응답에는 출처가 표시됩니다.
![](../images/0131141110786.png)

- 숫자 배지를 클릭하면 원문 확인
- 어떤 문서에서 정보를 가져왔는지 투명하게 표시
- 정보의 신뢰성 검증 가능

### 후속 질문 추천

AI 응답이 끝난 뒤, 자연스럽게 이어서 물어볼 만한 **후속 질문 몇 개가 자동으로 추천**됩니다. 추천 질문을 클릭하면 바로 입력창에 담겨 전송할 수 있어, 대화 흐름이 끊기지 않고 더 깊이 있는 탐색을 이어갈 수 있습니다.

## 웹 검색

AI가 실시간으로 웹을 검색하여 최신 정보를 제공합니다.

![](../images/0131141012236.png)

**작동 방식:**
1. 최신 정보가 필요한 질문 인식
2. 자동으로 웹 검색 수행
3. 검색 결과를 기반으로 답변 생성
4. 출처 링크 제공

**활용 예시:**
- "오늘 코스피 지수 알려줘"
- "최신 AI 트렌드가 뭐야"
- "이 회사 최근 뉴스 찾아줘"

## 대화 관리

### 채팅 검색

사이드바 상단의 검색 입력란에서 대화를 빠르게 찾을 수 있습니다.

**검색 방식:**
- **제목 검색**: 대화 제목에 포함된 키워드로 검색
- **내용 검색**: 대화 본문의 메시지 내용으로 검색
- **태그 검색**: `tag:태그명` 형식으로 특정 태그의 대화만 필터링

> 💡 **Tip**: `Ctrl + K` 단축키로 검색창에 빠르게 접근할 수 있습니다.

### 채팅 폴더

대화를 폴더로 분류하여 체계적으로 관리할 수 있습니다.

**폴더 만들기:**
1. 사이드바에서 **"새 폴더"** 버튼 클릭
2. 폴더 이름 입력
3. 필요에 따라 하위 폴더 생성 (폴더 중첩 지원)

**대화를 폴더로 이동:**
1. 대화 항목에서 우클릭 또는 메뉴 버튼 클릭
2. **"폴더로 이동"** 선택
3. 대상 폴더 선택

**폴더 관리:**
- 폴더 이름 변경
- 폴더 위치 이동 (상위 폴더 변경)
- 폴더 접기/펼치기
- 폴더 삭제

> 💡 **Tip**: 프로젝트별, 업무 유형별로 폴더를 만들면 수십 개의 대화도 쉽게 관리할 수 있습니다.

### 핀 고정

중요한 대화를 사이드바 상단에 고정할 수 있습니다.

1. 대화 항목에서 메뉴 버튼 클릭
2. **"핀 고정"** 선택
3. 고정된 대화는 사이드바 상단의 **핀 고정** 섹션에 표시

핀 고정을 해제하려면 동일한 메뉴에서 **"핀 해제"**를 선택합니다. 핀 고정된 대화는 폴더에 속한 대화와 별도로 항상 상단에 노출됩니다.

### 태그로 정리하기

대화에 태그를 추가하여 카테고리별로 분류할 수 있습니다.

**태그 추가:**
1. 대화 항목에서 메뉴 버튼 클릭
2. **"태그 추가"** 선택
3. 태그 이름 입력 (기존 태그 선택 또는 새 태그 생성)

**태그로 필터링:**
- 사이드바에서 태그를 클릭하면 해당 태그가 붙은 대화만 표시
- 검색창에 `tag:태그명`을 입력하여 필터링

**태그 자동 정리:**
- 태그에 연결된 마지막 대화가 삭제되면 해당 태그도 자동으로 삭제됩니다

**추천 태그 예시:**
- 📊 데이터분석
- 📝 보고서
- 💻 코딩
- 📧 이메일
- 🔍 조사

### 대화 아카이브

더 이상 활발하게 사용하지 않지만 보관하고 싶은 대화를 아카이브할 수 있습니다.

**아카이브 vs 삭제:**

| 동작 | 설명 | 복구 가능 |
|------|------|-----------|
| **아카이브** | 대화를 사이드바에서 숨기고 아카이브 목록으로 이동 | 언제든 복원 가능 |
| **삭제** | 대화를 완전히 제거 | 복구 불가 |

**아카이브 방법:**
1. 대화 항목에서 메뉴 버튼 클릭
2. **"아카이브"** 선택
3. 대화가 사이드바에서 사라지고 아카이브 목록으로 이동

**아카이브된 대화 확인:**
- 사이드바 하단의 **"아카이브된 채팅"** 메뉴에서 확인
- 아카이브된 대화를 다시 활성화하려면 **"아카이브 해제"** 클릭

**일괄 아카이브:**
- **"모든 대화 아카이브"** 기능으로 전체 대화를 한번에 아카이브할 수 있습니다

> 💡 **Tip**: 오래된 대화는 삭제 대신 아카이브하세요. 나중에 참고가 필요할 때 다시 찾아볼 수 있습니다.

### 임시 채팅

저장되지 않는 일회성 대화를 시작할 수 있습니다.

**임시 채팅 활성화:**
1. 모델 선택기 하단의 **"임시 채팅"** 토글을 활성화
2. URL에 `?temporary-chat=true` 파라미터가 추가됨
3. 이후 대화는 채팅 목록에 저장되지 않음

**임시 채팅의 특징:**
- 대화 기록이 서버에 저장되지 않음
- 사이드바 채팅 목록에 나타나지 않음
- 메시지 평가(좋아요/싫어요) 기능이 비활성화됨
- 메모리 추출이 수행되지 않음
- 브라우저를 닫거나 새 채팅을 시작하면 대화 내용이 사라짐

**활용 사례:**
- 민감한 정보를 포함한 일회성 질문
- 빠른 테스트나 실험
- 기록에 남기고 싶지 않은 대화

> 💡 **Tip**: 관리자가 임시 채팅 권한을 설정할 수 있으며, 조직 정책에 따라 임시 채팅을 강제하거나 비활성화할 수 있습니다.

### 대화 공유

동료와 대화 내용을 공유할 수 있습니다.

1. 채팅 메뉴에서 **"공유"** 클릭
2. 공유 링크 생성
3. 링크를 동료에게 전달
4. 링크를 통해 대화 내용 확인

## 음성 대화 (음성 통화)

실시간 음성으로 AI와 대화할 수 있습니다.

### 음성 통화 시작

1. 입력창 옆 **전화 버튼** 클릭
2. 마이크 권한 허용
3. 음성으로 대화 시작

**장점:**
- 핸즈프리 작업
- 자연스러운 대화
- 실시간 응답

### AI 아바타

AI 응답을 실감나는 아바타 캐릭터가 음성과 함께 전달합니다. Azure AI Speech 기반의 고품질 아바타로 마치 실제 사람과 대화하는 듯한 경험을 제공합니다.
![](../images/0131141226282.png)


**주요 기능:**

| 기능 | 설명 |
|------|------|
| **실시간 립싱크** | 음성에 맞춰 자연스러운 입 모양 동기화 |
| **다양한 캐릭터** | 여러 아바타 중 선택 가능 |
| **표정 애니메이션** | 대화 내용에 맞는 자연스러운 표정 |
| **고화질 렌더링** | HD급 영상 품질 |

**활용 사례:**
- 🎓 **교육/트레이닝**: 대화형 학습 콘텐츠
- 🏢 **키오스크**: 무인 안내 시스템
- 💼 **프레젠테이션**: AI 발표자 활용
- 🎧 **접근성**: 시각적 피드백이 필요한 사용자

**아바타 사용 방법:**
1. 음성 통화 모드 시작
2. 설정에서 아바타 활성화
3. 원하는 아바타 캐릭터 선택
4. AI와 대화 시 아바타가 응답

> 💡 **Tip**: 아바타 기능은 Azure AI Speech 서비스와 연동되어 자연스러운 음성과 영상을 제공합니다.

## 메모리 시스템

Cloosphere는 대화 내용에서 중요한 정보를 자동으로 추출하고 기억하는 메모리 시스템을 제공합니다. 메모리를 통해 AI는 사용자의 선호도, 업무 맥락, 기술 스택 등을 기억하여 점점 더 개인화된 응답을 제공합니다.

<!-- 스크린샷: 메모리 관리 UI
     파일명: images/chat-memory-management.png
-->

### 메모리 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **장기 메모리** | 대화 전반에 걸쳐 유지되는 사용자 정보 | "Python과 TypeScript를 주로 사용합니다" |
| **단기 메모리** | 현재 대화 세션 내에서 참조되는 정보 | "이번 프로젝트는 React 기반입니다" |

### 자동 메모리 추출

AI가 대화 중 사용자의 선호도, 업무 스타일, 프로젝트 정보 등을 자동으로 감지하고 메모리에 저장합니다. 자동 추출된 메모리에는 **Auto** 배지가 표시되어 수동으로 추가한 메모리와 구분됩니다.

**자동 추출 대상:**
- 사용자 선호도 및 스타일
- 프로젝트 및 기술 스택 정보
- 업무 관련 규칙 및 컨벤션
- 반복적으로 언급되는 요구사항

**추출 과정:**
1. 대화 완료 후 백그라운드에서 메모리 추출 파이프라인이 실행됩니다
2. AI가 대화 내용을 분석하여 기억할 만한 팩트를 식별합니다
3. 신뢰도 기준을 충족하는 정보만 메모리로 저장됩니다 (관리자가 신뢰도 임계값 설정 가능)
4. 동일 대화에서 중복 추출을 방지하기 위한 디바운스(5분) 메커니즘이 적용됩니다

> 💡 **Tip**: 임시 채팅에서는 메모리 추출이 수행되지 않습니다.

### Profile Consolidation (프로필 통합)

시간이 지남에 따라 축적된 메모리는 자동으로 **프로필 통합(Profile Consolidation)** 과정을 거칩니다. 이는 개별 팩트들을 구조화된 프로필 문서로 통합하는 과정입니다.

**통합 트리거 조건:**
- 마지막 통합 이후 **24시간**이 경과하고, **5개 이상의 새로운 팩트**가 축적된 경우

**통합 과정:**
1. 모든 개별 메모리 팩트를 수집합니다
2. AI가 중복되거나 오래된 정보를 정리합니다
3. 핵심 정보를 카테고리별로 정리한 프로필 문서를 생성합니다:
   - **역할 및 업무** -- 직무, 담당 업무
   - **기술 선호도** -- 사용 언어, 프레임워크, 도구
   - **진행 중인 프로젝트** -- 현재 작업 중인 내용
   - **제약 조건 및 요구사항** -- 업무 규칙, 컨벤션
   - **커뮤니케이션 스타일** -- 선호하는 응답 방식
4. 통합된 프로필은 모든 후속 대화에서 AI의 컨텍스트로 활용됩니다

### 메모리가 대화를 개선하는 방식

메모리 시스템은 벡터 DB 기반의 **시맨틱 검색**을 통해 현재 대화와 관련된 과거 정보를 자동으로 찾아 활용합니다.

**예시:**
- "보고서 작성해줘" → 이전에 저장된 선호 보고서 형식을 자동으로 적용
- "API 설계해줘" → 사용자가 선호하는 언어와 프레임워크를 반영
- "코드 리뷰해줘" → 팀의 코딩 컨벤션을 고려하여 피드백 제공

### 메모리 관리

**설정 > 개인화 > 메모리**에서 저장된 메모리를 확인하고 관리할 수 있습니다.

<!-- 스크린샷: 메모리 목록 및 편집 UI
     파일명: images/chat-memory-list.png
-->

**관리 기능:**

| 기능 | 설명 |
|------|------|
| **목록 확인** | 저장된 모든 메모리를 날짜순으로 확인. Auto/Manual 배지로 출처 구분 |
| **수동 추가** | 직접 메모리를 추가하여 AI에게 기억시킬 정보 입력 |
| **개별 편집** | 기존 메모리의 내용을 수정 |
| **개별 삭제** | 잘못 저장된 메모리를 선택적으로 삭제 |
| **전체 초기화** | 모든 메모리를 한번에 삭제하고 처음부터 다시 시작 |
| **활성화/비활성화** | 메모리 기능 자체를 켜거나 끌 수 있음 |

> 💡 **Tip**: 잘못 저장된 메모리가 있다면 개별적으로 삭제하여 AI 응답 품질을 유지하세요. 메모리가 너무 많이 쌓이면 프로필 통합이 자동으로 정리해 주지만, 명백히 잘못된 정보는 수동으로 삭제하는 것이 좋습니다.

## Human-in-the-Loop 인터랙티브 위자드

복잡한 요청을 처리할 때, AI가 사용자에게 추가 정보를 요청하는 인터랙티브 위자드 기능을 제공합니다.

<!-- 스크린샷: Human-in-the-Loop 위자드 예시
     파일명: images/chat-hitl-wizard.png
-->

### 작동 방식

1. 사용자가 복잡한 요청을 전송
2. AI가 추가 정보가 필요하다고 판단
3. **인터랙티브 위자드** 형태로 추가 질문을 표시
4. 사용자가 선택 또는 입력으로 응답
5. AI가 수집된 정보를 바탕으로 최종 응답 생성

### 위자드 요소

| 요소 | 설명 |
|------|------|
| **선택 버튼** | 미리 정의된 옵션 중 선택 |
| **텍스트 입력** | 자유 형식 추가 정보 입력 |
| **확인/취소** | 위자드 진행 또는 건너뛰기 |

**활용 사례:**
- 에이전트가 데이터베이스 쿼리 전 테이블/컬럼 확인
- 보고서 생성 시 포맷, 범위, 대상 등 세부 사항 수집
- 모호한 요청에 대한 의도 명확화

> 💡 **Tip**: 위자드를 건너뛰면 AI가 기본값으로 진행합니다. 정확한 결과를 원하면 위자드에 응답하세요.

## 개인 설정

설정 모달에서 채팅 환경을 커스터마이즈할 수 있습니다.

![](../images/0131141332052.png)

### 인터페이스 설정

- **테마**: 라이트/다크/시스템
- **언어**: 한국어, English, 日本語
- **와이드스크린**: 넓은 화면 모드

### 오디오 설정

- **STT 엔진**: 음성 인식 엔진 선택
- **TTS 엔진**: 음성 합성 엔진 선택
- **음성**: 선호하는 음성 선택

### 개인화

- **메모리**: AI가 기억할 정보 저장 (메모리 시스템 참조)
- **컨텍스트**: 모든 대화에 적용될 배경 정보
