# 프로젝트 (Projects)

> 프로젝트는 개인 문서 공간입니다. 지식 베이스를 기반으로 문서를 정리하고, 프로젝트 컨텍스트 내에서 AI와 대화할 수 있습니다. 팀원에게 프로젝트를 공유하여 같은 문서를 함께 활용하세요.

---

## 프로젝트란?

프로젝트는 **지식 베이스를 래핑한 개인 문서 공간**입니다. 채팅에서 프로젝트를 선택하면 해당 프로젝트의 문서만을 참조하여 AI가 답변합니다.

### 프로젝트 vs 지식 베이스

| 구분 | 프로젝트 | 지식 베이스 |
|------|---------|------------|
| **목적** | 개인/팀 문서 공간 | 에이전트용 지식 저장소 |
| **소유** | 개인 소유, 선택적 공유 | 워크스페이스 공유 |
| **접근** | 사이드바에서 바로 접근 | 에이전트에 연결하여 사용 |
| **채팅** | 프로젝트 내 전용 채팅 | 에이전트 채팅에서 참조 |

### 활용 사례

- **업무 문서 관리**: 프로젝트별 자료를 모아두고 AI에게 질문
- **팀 협업**: 부서 공용 문서를 프로젝트로 묶어 팀원과 공유
- **리서치**: 연구 자료를 정리하고 AI와 대화하며 분석

## 프로젝트 목록

사이드바 하단의 **프로젝트** 섹션에서 내 프로젝트와 공유받은 프로젝트를 확인할 수 있습니다.

---

## 프로젝트 생성

### 1단계: 새 프로젝트 만들기

사이드바의 프로젝트 섹션에서 **"+"** 버튼을 클릭합니다.

### 2단계: 기본 정보 입력

| 필드 | 설명 | 예시 |
|------|------|------|
| **이름** | 프로젝트 표시 이름 | "2026 마케팅 전략" |
| **설명** | 프로젝트 용도 설명 | "Q1 마케팅 캠페인 관련 자료" |

### 3단계: 저장

**"생성"** 버튼을 클릭하면 프로젝트가 생성됩니다.

---

## 파일 관리

### 파일 업로드

프로젝트 상세 화면에서 파일을 업로드합니다.

**지원 파일 형식:**
- PDF, DOCX, PPTX, XLSX, CSV
- TXT, MD (마크다운)
- 기타 (LibreOffice PDF 변환 활성화 시)

**업로드 방법:**
1. **드래그 앤 드롭**: 파일을 프로젝트 영역에 끌어다 놓기
2. **파일 선택**: 업로드 버튼 클릭 후 파일 선택
3. **클라우드 스토리지**: Google Drive, OneDrive, SharePoint에서 직접 가져오기

### 파일 목록

업로드된 파일은 프로젝트 상세 화면의 **파일** 탭에서 확인할 수 있습니다.

| 정보 | 설명 |
|------|------|
| **파일명** | 업로드된 파일 이름 |
| **크기** | 파일 크기 |
| **상태** | 처리 상태 (업로드 중, 처리 완료, 오류) |
| **업로드일** | 파일 업로드 날짜 |

### 파일 삭제

파일 우측 메뉴에서 **삭제**를 클릭합니다. 삭제된 파일은 벡터 DB에서도 함께 제거됩니다.

## 프로젝트 공유

프로젝트를 특정 사용자에게 공유할 수 있습니다.

### 공유 방법

1. 프로젝트 상세 화면에서 **설정** 탭으로 이동
2. **"공유"** 섹션에서 사용자 검색
3. 공유할 사용자 선택
4. 저장

### 공유 특성

- 공유받은 사용자에게 **독립적인 복사본**이 생성됩니다
- 원본 프로젝트의 파일이 복사되어 각자 독립적으로 관리됩니다
- 공유 후 각 사용자가 자신의 프로젝트를 자유롭게 수정할 수 있습니다

---

## 프로젝트에서 채팅

### 프로젝트 컨텍스트 채팅

프로젝트를 선택한 상태에서 채팅을 시작하면, 해당 프로젝트의 문서만을 참조하여 AI가 답변합니다.

**사용 방법:**
1. 사이드바에서 프로젝트 클릭
2. 채팅 입력창에 질문 입력
3. AI가 프로젝트 문서를 검색하여 답변

**예시:**
```
Q: 이번 분기 마케팅 예산 계획은?
A: 프로젝트 문서를 확인한 결과, 이번 분기 마케팅 예산은...
   [출처: Q1_마케팅_계획.pdf, 12페이지]
```

### 프로젝트 채팅 관리

프로젝트 내에서 진행한 채팅은 프로젝트 상세 화면의 **채팅** 탭에서 확인할 수 있습니다. 메인 사이드바의 채팅 목록에는 프로젝트 채팅이 표시되지 않아 깔끔하게 관리됩니다.

---

## 프로젝트 설정

프로젝트 상세 화면의 **설정** 탭에서 프로젝트 정보를 수정할 수 있습니다.

| 항목 | 설명 |
|------|------|
| **이름** | 프로젝트 이름 변경 |
| **설명** | 프로젝트 설명 수정 |
| **공유** | 사용자 공유 관리 |

---

## 프로젝트 삭제

프로젝트를 삭제하면 포함된 모든 파일과 채팅 기록이 함께 삭제됩니다.

> **주의:** 삭제된 프로젝트는 복구할 수 없습니다. 공유받은 사용자의 복사본에는 영향을 주지 않습니다.

## 신규 기능 (2025년 3월 이후)

### 데이터 분석 프로젝트 (Code Interpreter)

Jupyter 노트북 기반의 코드 실행 환경을 프로젝트 내에서 사용할 수 있습니다. AI가 Python 코드를 작성하고 실행하여 데이터 분석, 시각화, 파일 변환 등을 수행합니다.

<!-- 스크린샷: 데이터 분석 프로젝트 화면
     - Jupyter 노트북 실행 결과 (차트, 표 등)
     - AI와의 대화형 코드 실행
     파일명: images/projects-code-interpreter.png
-->

**주요 기능:**
- Python 코드 실행을 통한 데이터 분석 및 시각화
- 업로드된 파일(CSV, XLSX 등)을 직접 분석
- 차트, 그래프, 통계 결과를 대화 내에서 확인
- 분석 결과 파일 다운로드

**활용 예시:**
```
Q: 매출_데이터.csv 파일을 분석하고 월별 추이 그래프를 만들어 주세요
A: 매출 데이터를 분석했습니다. 월별 추이는 다음과 같습니다:
   [차트: 월별 매출 추이 그래프]
   
   주요 발견:
   - 3월에 전월 대비 23% 매출 증가
   - 하반기 평균 매출이 상반기 대비 15% 높음
```

### 프로젝트 파일 자동 재마운트

Jupyter 커널이 재시작되더라도 프로젝트에 업로드된 파일이 자동으로 다시 마운트됩니다. 커널 재시작 후 파일을 다시 업로드할 필요 없이 바로 작업을 계속할 수 있습니다.

<!-- 스크린샷: 커널 재시작 후 파일이 유지된 화면
     파일명: images/projects-auto-remount.png
-->

**동작 방식:**
- Jupyter 커널 재시작 시 프로젝트 파일이 작업 디렉토리에 자동 복원
- 세션 간 파일 경로가 일관되게 유지
- 별도의 재업로드 작업 불필요

### 커널 보안 샌드박스

프로젝트별로 독립된 워크스페이스가 격리되어 실행됩니다. 각 프로젝트의 코드 실행 환경은 다른 프로젝트와 완전히 분리되어 보안과 안정성을 보장합니다.

<!-- 스크린샷: 프로젝트별 격리된 실행 환경 다이어그램
     파일명: images/projects-sandbox.png
-->

**보안 특징:**
- 프로젝트별 독립 워크스페이스로 파일 시스템 격리
- 다른 프로젝트의 데이터에 접근 불가
- 리소스 사용량 제한으로 시스템 안정성 확보

### 3단계 도구 + 파일 메타데이터 강화

프로젝트의 도구 기능이 3단계로 확장되었으며, 파일 메타데이터가 강화되었습니다.

<!-- 스크린샷: 프로젝트 도구 및 파일 메타데이터 설정 화면
     파일명: images/projects-tools-metadata.png
-->

**도구 3단계:**

| 단계 | 설명 |
|------|------|
| **1단계: 기본 도구** | 파일 검색 및 문서 참조 |
| **2단계: 분석 도구** | Code Interpreter 기반 데이터 분석 |
| **3단계: 확장 도구** | 외부 API 연동 및 고급 처리 |

**파일 메타데이터 강화:**
- 파일별 상세 메타데이터 확인 (파일 유형, 크기, 페이지 수, 처리 상태 등)
- 메타데이터 기반 파일 정렬 및 필터링
- 처리 이력 추적

## FAQ

**Q: 프로젝트에 파일 수 제한이 있나요?**
> 시스템 설정에 따라 다릅니다. 일반적으로 파일 수 제한은 없지만, 파일 크기 제한은 관리자 설정을 따릅니다.

**Q: 프로젝트를 공유하면 실시간으로 동기화되나요?**
> 아니요, 공유 시 독립 복사본이 생성됩니다. 원본을 수정해도 공유받은 프로젝트에는 반영되지 않습니다.

**Q: 기존 지식 베이스를 프로젝트로 변환할 수 있나요?**
> 직접 변환은 불가합니다. 프로젝트를 생성한 후 파일을 다시 업로드하세요.
